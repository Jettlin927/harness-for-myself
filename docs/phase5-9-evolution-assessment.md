# HAU 全面评估与演进路线 — Phase 5-9

> 评估时间：2026-03-25，基于 Phase 1-4 + P0 修复完成后的项目状态

## 一、项目定位

HAU 是一个 **5,575 行代码的单 agent 编程 harness**（3,200 行核心 + 2,375 行测试），已完成 Phase 1-4 + P0 修复。能力：用 Claude API 自主读代码、搜索代码、编辑文件、创建文件、跑命令，具备流式输出、三级权限、项目感知、会话持久化。

**一句话评价：工程框架 A-，实际编程效果 C+。**

框架扎实（schema 校验、快照恢复、错误策略、131 个测试），但 agent 的「智商」受限于过于简短的 system prompt 和过浅的上下文管理。

## 三、演进路线

### Phase 5: Agent 智能提升（让 agent「会用」工具）

| 任务 | 改动文件 | 工作量 | 影响 |
|------|---------|--------|------|
| 5a. 重写 system prompt | llm.py | 小 | **极高** — 性价比之王 |
| 5b. max_history_turns 8→20 | agent.py | 极小 | 高 |
| 5c. 工具输出截断 | memory.py | 小 | 高 |
| 5d. turn 编号注入 | anthropic_llm.py | 极小 | 中 |

**5a 详细设计 — System Prompt 应包含：**
1. 角色定义：你是一个编程 agent
2. 工具使用策略：搜索→理解→修改→验证
3. 错误恢复：失败后分析原因，不重复相同操作
4. 最小改动原则
5. 验证习惯：修改后跑测试/lint
6. 上下文标记：用 constraint:/todo:/evidence: 前缀

## 四、推荐执行顺序

```
Phase 5 (agent 智能)  ← 最高优先级
    ↓
Phase 6 (可靠性)      ← 部分可与 Phase 5 并行
    ↓
Phase 8 (发布 v0.1)
    ↓
Phase 7 (自验证 v0.2)
    ↓
Phase 9 (架构 v1.0)
```

**关键洞察：** Phase 5a（system prompt 重写）是当前整个项目性价比最高的一步。prompt 就是 agent 的「大脑」。
